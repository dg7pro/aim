ionCube

Maian Support is an encrypted system using the PHP encoder ionCube. This means that many of the main processing files within the script have been encrypted to bytecode to prevent 3rd party modifications, such as illegal copyright removal. If any of these files are changed the system will fail to load. Files encoded using this system will only run on a server with the ionCube Loaders installed.

Installation of these loaders is a very simple process and should only take a few moments. In most cases ionCube loaders are available as default on most shared hosting environments. If the system loads ok, you won`t need to do anything else.

ionCube Loaders

This script requires the installation of ionCube loaders on the server where the script is to be installed.

The very latest loaders can be downloaded from the ionCube website.

Installing ionCube Loaders

To install the PHP loaders you first need to find out which loaders you need. This will be based on your server and PHP version. PHP v5.2.5 or later is recommended, but not essential. ionCube provide an easy to use PHP file to detail the procedure you need to follow to get the loaders to work. This is as follows:

1. Upload the 'docs/ioncube/loader-wizard.php' file to anywhere on your server.
2. Access this file in your browser and follow the instructions on screen.

If you have the loaders installed, you don`t need to do anything else. You can proceed with the Maian Support installation.

If you don`t, follow the instructions on screen. Changes to your PHP.ini file will be required in most cases. If you don`t understand this, contact your host and ask them to install the latest loaders for ionCube.

If you are on a shared server, refer to the notes in the following file:

docs/ioncube/README.txt

If you run your own server, you`ll need to do the installation yourself. This is a simple procedure as detailed with running the wizard.

IMPORTANT: Encoded System Files

Encoded PHP files should NOT be modified in any way. If you do edit a file and it fails to load, please re-download the script and replace the corrupted file. Only the following are encoded:

index.php
control/system/core/*
admin/index.php
admin/control/system/core/*
