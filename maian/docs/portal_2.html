Overview

The main core of an help desk system is the opening of tickets.

Creating New Ticket

On the main help desk page, visitors will click 'Open New Support Ticket' to launch the ticket creation page:



Your Name

Name of person submitting ticket. Pre-filled if logged in already.

E-Mail Address

Email of person submitting ticket. Pre-filled if logged in already.

Department

Department best suited to ticket query.

Subject

Brief description of issue.

Comments

More detailed description of issue.

BBCode Text Formatting

BB Code is a collection of formatting tags that are used to change the look of text. BB Code is based on a similar principal to HTML and allows visitors to safely format text without danger of upsetting any of the system layout via certain html tags. If enabled, most popular tags are shown as buttons. Clicking the ? button shows more tags and information.



Note that BBCode is not parsed in email templates.

Priority

Priority selection. Either low, medium or high.

Custom Fields

Custom data may be requested by admin. This may be in the form of text options or checkboxes and may also be required.

Attachment(s)

Attachments may be enabled with tickets via admin. Restrictions may also be imposed on file size and file types. Note that the free version allows only a single attachment with each ticket and/or reply. If multiple attachments are available, boxes can be added/removed via the links provided:

Spam Prevention

If enabled, spam prevention is available via Googles reCaptcha service. More info here.

E-Mails Sent

When a new ticket is created, emails are sent to visitor to confirm ticket and also to any relevant admin users. If this is the first ticket from this email address, a portal password is auto created, which can be changed when the visitor logs in.

Email templates can be edited if you wish. More info here.
